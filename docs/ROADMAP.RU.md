## 2. Детальный Roadmap с описанием новых функций

### Фаза 1.3.x — Удобство и эргономика (март – июнь 2026)

**Новые функции:**

1. **`FromSlice[T](slice []T) *Set[T]`**  
   Создаёт новое множество из слайса. Дубликаты автоматически удаляются.  
   *Описание:* Эквивалент `NewSet(slice...)`, но более читаемый и удобный для цепочек.

2. **`FromSliceOrdered[T Ordered](slice []T) *OrderedSet[T]`**  
   Создаёт `OrderedSet` из слайса.  
   *Описание:* Полезно, когда нужно сразу получить упорядоченное множество.

3. **`FromIter[T](seq iter.Seq[T]) *Set[T]`** и аналоги для Sync/Ordered  
   Создаёт множество из итератора `iter.Seq`.  
   *Описание:* Позволяет легко конвертировать любой итерируемый источник (канал, генератор и т.д.) в множество.

4. **`FromMapKeys[T comparable, V any](m map[T]V) *Set[T]`**  
   Создаёт множество из ключей карты.  
   *Описание:* Очень удобно при работе с `map` как с множеством ключей.

5. **`FromChan[T](ch <-chan T) *Set[T]`** (с опциональным контекстом)  
   Собирает все значения из канала в множество.  
   *Описание:* Поддерживает `context.Context` для отмены.

6. **SmallSet Optimization** (внутренняя)  
   Автоматически использует массив вместо `map` при размере ≤ 12 элементов.  
   *Описание:* Значительно снижает аллокации и улучшает производительность для маленьких множеств.

7. **`PopMin() (T, bool)` / `PopMax() (T, bool)`** в `OrderedSet` и `SyncOrderedSet`  
   *Описание:* Возвращает и удаляет минимальный/максимальный элемент. `bool` показывает, было ли множество непустым.

8. **`RemoveMin()` / `RemoveMax()`**  
   *Описание:* Удаляет минимальный/максимальный элемент, если он существует (без возврата значения).

9. **`Collect[T](seq iter.Seq[T]) *Set[T]`**  
   *Описание:* Собирает все элементы из итератора в новое множество (удобная альтернатива `FromIter`).

10. **`Fold[T, U any](s ISet[T], initial U, fn func(U, T) U) U`**  
    *Описание:* Левая свёртка (fold left). Более общая версия `Reduce`.

**Целевая версия:** **v1.3.0** — май 2026

### Фаза 1.4.x – 1.5.x — Производительность и надёжность (июль – декабрь 2026)

**Новые функции:**

11. **`MinSafe() (T, bool)` / `MaxSafe() (T, bool)` / `SumSafe() (T, bool)`**  
    *Описание:* Возвращают значение + флаг существования. Решают проблему zero-value на пустом множестве.

12. **`TryAdd(element T) bool`** / **`TryRemove(element T) bool`**  
    *Описание:* Возвращают `true`, если элемент был действительно добавлен/удалён (полезно для атомарных проверок).

13. **`ForEachWithIndex[T](s ISet[T], fn func(int, T))`**  
    *Описание:* Вызывает функцию для каждого элемента с его индексом в текущей итерации (порядок недетерминирован).

14. **`TakeWhile[T](s ISet[T], predicate func(T) bool) ISet[T]`**  
    *Описание:* Возвращает новое множество, содержащее элементы до первого, не удовлетворяющего предикату.

15. **`DropWhile[T](s ISet[T], predicate func(T) bool) ISet[T]`**  
    *Описание:* Возвращает новое множество, начиная с первого элемента, не удовлетворяющего предикату.

16. **`Zip[T, U any](s1 ISet[T], s2 ISet[U]) iter.Seq2[T, U]`**  
    *Описание:* Ленивый zip двух множеств (по минимальному размеру).

17. **Кэширование экстремальных значений в `OrderedSet`**  
    *Описание:* `Min`/`Max`/`Sum` кэшируются и инвалидируются только при изменении множества.

18. **Оптимизация `ToSlice()`** — уменьшение аллокаций при частых вызовах.

**Целевая версия:** **v1.5.0** — ноябрь 2026 (LTS)

### Фаза 1.6.x – 1.8.x — Расширение возможностей (2027)

**Новые типы и функции:**

19. **`BitSet`** (новый тип)  
    *Описание:* Сверхбыстрое zero-allocation множество для целых чисел в диапазоне 0..N. Поддерживает битовые операции (`And`, `Or`, `Xor`, `Not`).

20. **`DenseIntSet`**  
    *Описание:* Оптимизированное множество для плотных диапазонов целых чисел (лучше `BitSet` при разреженных данных).

21. **`TreeOrderedSet[T Ordered]`**  
    *Описание:* Упорядоченное множество на основе сбалансированного дерева (Red-Black или AVL). Гарантирует порядок вставки или сортировки, O(log n) для всех операций.

22. **`NewSetWithHasher[T](hasher func(T) uint64) *Set[T]`**  
    *Описание:* Позволяет использовать собственную хеш-функцию (полезно для структур с нестандартным равенством).

23. **Контекстно-зависимые методы:**
    - `ForEachWithContext(ctx context.Context, action func(T)) error`
    - `AnyWithContext`, `AllWithContext`, `FilterWithContext`, `ReduceWithContext` и т.д.
    *Описание:* Поддержка отмены длинных операций.

24. **`PowerSet[T](s ISet[T]) ISet[ISet[T]]`**  
    *Описание:* Возвращает множество всех подмножеств данного множества.

25. **`CartesianProduct[T, U any](s1 ISet[T], s2 ISet[U]) ISet[struct{A T; B U}]`**  
    *Описание:* Декартово произведение двух множеств.

26. **`SymmetricDifferenceWith[T](s ISet[T], other ISet[T], eq func(T,T) bool) ISet[T]`**  
    *Описание:* Симметрическая разность с кастомным равенством.

27. **Сериализация:**
    - Поддержка `encoding/gob`
    - Отдельный пакет `go-set/codec` для msgpack / protobuf (опционально)

28. **Интроспекция:**
    - `Capacity() int`
    - `LoadFactor() float64`
    - `AllocatedMemoryEstimate() uintptr`
    - `StringWithOptions(opts StringOptions)`

**Целевые версии:**
- **v1.6.0** — март 2027
- **v1.7.0** — июль 2027
- **v1.8.0** — декабрь 2027

### Фаза 2.0.0 — Major Release (конец 2027 – начало 2028)

**Новый модуль:** `go-set/v2`

**Ключевые изменения:**
- Переработанная иерархия интерфейсов (`ICollection`, `ISet`, `IOrderedSet`, `ISortedSet`)
- `OrderedSet` с выбором режима порядка (`InsertionOrder` / `SortedOrder`)
- Встроенная поддержка `context.Context` во всех операциях
- Изменение поведения на пустых множествах (возврат ошибок вместо zero-value)
- Унифицированные конструкторы
- Возможный отказ от глубокого embedding в пользу composition
- Полноценный `MIGRATION.md` + инструменты миграции

---

### Фаза 3.0.0 — Коллекционная экосистема (2028–2029)

**Глобальная цель:** Создать полноценное семейство коллекций.

**Планируемые пакеты:**
- `set`, `orderedset`, `map`, `list`, `deque`, `priorityqueue`, `ringbuffer`, `bitset`
- Общий пакет `collections` с утилитами

**Ключевые возможности:**
- Единый дизайн API и производительности
- Поддержка arena allocation и zero-copy
- Генерация кода через `go generate`
- Несколько бэкендов (map / tree / bitset / concurrent)
- Интеграция с будущими фичами Go

# Contributing to go-set

Спасибо за интерес к проекту **go-set**. Любые улучшения и вклад приветствуются.

## Как внести вклад

### 1. Клонируйте репозиторий

```bash id="cl7p1k"
git clone https://github.com/encomers/go-set.git
cd go-set
```

### 2. Создайте ветку

Всегда работайте в отдельной ветке:

```bash id="zq8v2m"
git checkout -b feature/your-feature-name
```

### 3. Внесите изменения

* Пишите простой и понятный код
* Следуйте текущему стилю проекта
* Избегайте лишних абстракций
* Не добавляйте внешние зависимости

### 4. Напишите тесты

* Добавляйте тесты для новой функциональности
* Обновляйте существующие тесты при изменении поведения
* Убедитесь, что все тесты проходят:

```bash id="4b2c1u"
go test ./...
```

### 5. Запустите бенчмарки (по желанию)

Если изменения влияют на производительность:

```bash id="x7m2pq"
go test -bench=. -benchmem
```

### 6. Сделайте коммит

Используйте понятные сообщения:

```bash id="k2r9vs"
git commit -m "Add: новая функциональность X"
git commit -m "Fix: исправление проблемы Y"
```

### 7. Откройте Pull Request

* Опишите, что изменено и зачем
* Приложите бенчмарки при необходимости
* Делайте PR небольшими и сфокусированными

---

## Рекомендации по коду

* Используйте возможности Go 1.23+
* Следуйте идиоматическому Go
* Не ломайте публичный API без необходимости
* Сохраняйте обратную совместимость

---

## Принципы проекта

* **Без внешних зависимостей**
* **Высокая производительность**
* **Минимальные аллокации**
* **Простой и предсказуемый API**

---

## Сообщение об ошибках

Если вы нашли баг или хотите предложить улучшение:

1. Проверьте, нет ли уже похожей задачи
2. Опишите проблему максимально ясно
3. По возможности приложите пример
